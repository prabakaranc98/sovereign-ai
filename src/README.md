# src — the lab bench

Where the system gets built. Not scattered experiments — **one cohesive system**, grown over the run, plus the small instruments needed to answer questions along the way. Every significant decision about it must survive interrogation from five lenses at once: research, engineering, design, product, human-centeredness (see SCRATCHPAD.md, "One system, five lenses").

Rules of the bench (from [research-engineering.md](../research-engineering.md)):
- Every build traces back to a scratchpad question that existed **before** the build. No question upstream → no build.
- Artifacts are instruments, not products — built to settle a question, kept only if they keep earning it.
- Negative results are results; they get written up like everything else.

Empty until a question demands a build.
